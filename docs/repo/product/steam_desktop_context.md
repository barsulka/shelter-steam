# Steam/Desktop Product Context

Дата синхронизации: 2026-06-24

Этот документ фиксирует repo-facing контекст для разработки Steam/Desktop-ветки Shelter. Источники: Google Drive `Shelter/00_START_HERE/00_PROJECT_INDEX.md`, `01_CURRENT_STATUS`, `02_DECISIONS`, `02_PRODUCTS/01_STEAM_DESKTOP/STEAM_DESKTOP__Game_Designer_Session_Brief`.

## Product Boundary

Steam/Desktop — отдельная Windows/macOS игра в Godot. Не смешивать её с Browser Extension:

- нет Chrome new-tab layout;
- нет browser search bar;
- нет sponsorship/ad block;
- нет rewarded ads;
- нет рекламной монетизации как core loop;
- нет extension-specific UX.

Browser Extension D-008 остаётся соседней продуктовой веткой и не является задачей этого репозитория.

## D-009 Core Structure

Steam/Desktop больше не считается классической фермой, ужатой в полоску.

Принята базовая структура: **горизонтальный собачий производственный кооператив** в sidescroll always-on-top полоске.

Основная формула:

- cozy idle production strip + dog community sim.

Игровое ядро:

- собаки живут в уютном кооперативе;
- выращивают или получают мирные ингредиенты;
- готовят корм в маленьком заводике;
- фасуют корм;
- складируют мешки;
- отправляют фургон в партнёрские приюты;
- игрок мягко организует систему: строит зоны, задаёт приоритеты, выбирает рецепты, назначает собак, улучшает процессы, украшает комнаты и наблюдает за медленной физической работой.

Базовая производственная линия:

```text
сад/теплица -> кладовая ингредиентов -> кухня -> миксер -> сушилка/пресс -> фасовка -> склад мешков -> погрузка/фургон
```

Ферма остаётся источником мирных ингредиентов, но ядро Steam/Desktop-версии — production strip / кооператив, потому что это лучше работает в горизонтальной нижней полоске и даёт видимые процессы.

## Player Role

Игрок не микроменеджит каждое действие. Он:

- строит и расширяет зоны;
- задаёт производственные приоритеты;
- выбирает рецепты;
- назначает подходящих собак;
- улучшает процессы;
- украшает комнаты;
- запускает и наблюдает поставки в приюты.

Система должна быть интересной при наблюдении, но не требовать постоянного активного управления.

## D-010 Dog Traits

Собака — персонаж, а не набор оптимизируемых статов.

Любая система собак, навыков, ролей, комнат, особенностей и прогрессии должна различать два слоя:

- врождённые / неизменяемые особенности: собака появляется с ними; это часть её личности; игрок не может удалить, заменить, "перековать" или стереть их;
- изменяемые / экипируемые / приобретённые особенности: предметы, уют комнаты, обучение, любимые инструменты, временные привычки, тапочки, ошейники, рабочие аксессуары, декорные эффекты.

Изменяемые особенности могут усиливать или оттенять врождённую индивидуальность собаки, но не стирать её.

## Resources And Tone

Производство корма использует мирные или абстрактные ресурсы:

- овощи;
- крупы;
- рис;
- овёс;
- травы;
- ягоды;
- вода;
- витамины;
- упаковка;
- белковые кубики;
- бульонные капсулы;
- протеин-пакеты;
- поставки партнёров;
- ветеринарная смесь.

Запрещены:

- бои;
- PvP;
- арены;
- боссы;
- монстры;
- paid gacha;
- рекламная монетизация в Steam;
- aggressive FOMO;
- guilt pressure;
- манипулятивная благотворительность;
- скотобойня;
- разделка животных;
- кровь;
- реалистичная мясная производственная цепочка;
- вайб эксплуатации "собак-рабочих".

Благотворительная миссия: готовый корм спокойно уезжает в партнёрские приюты. Игрок видит не страдание, а уютное собачье сообщество, которое помогает другим.

## Technical Spikes To Keep Next

После этой синхронизации следующие технические spike-задачи для Godot:

- прозрачное окно Godot;
- always-on-top toggle;
- click-through для пустых зон, если возможно;
- скрытие интерфейса одной кнопкой;
- нижняя горизонтальная полоса;
- zones/blocks grid;
- горизонтальный скролл;
- placeholder dog agent walking;
- первая production loop-заглушка: ingredient crate -> mixer -> food bag -> van.

При любых spike-задачах сохранять Windows/macOS scope, performance budget и документацию в `docs/repo/dev/` + `docs/repo/status/CODEX_STATUS.md`.
