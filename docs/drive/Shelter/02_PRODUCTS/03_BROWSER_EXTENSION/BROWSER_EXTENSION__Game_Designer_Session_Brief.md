# BROWSER_EXTENSION — Game Designer Session Brief

Дата: 2026-06-24  
Роль будущего чата: Game Designer Browser Extension / Idle Farm Designer.  
Рабочее название продукта: Shelter New Tab Farm / Dog Farm New Tab / Feed Dogs Extension.

## Role boundary note

Game Designer в этой сессии отвечает за core loop, механики, ресурсы, production chains, idle/offline progress, dog progression, player goals, retention, pacing, ads-as-gameplay constraints и UX-логику коротких new-tab визитов.

Game Designer может формулировать visual requirements только как gameplay constraints: какие действия, состояния, зоны и игровые сущности должны быть видимы и различимы. Game Designer не пишет art prompts, не выбирает palette/style, не проектирует art bible и не определяет asset pipeline. Вопросы visual direction, style board, prompts, readability, UI look и production asset style передаются Art Director.

## 1. Контекст проекта Shelter

Shelter — группа добрых приложений/игр вокруг собак, заботы, помощи и регулярного возвращения пользователя.

Общие запреты:  
- никаких боёв, PvP, арен, боссов, монстров;  
- никакой агрессивной соревновательности;  
- никакой манипулятивной благотворительности;  
- никакого давления на пользователя через вину;  
- никакой скотобойни, разделки животных, жестоких мясных производственных цепочек.

Тон: тёплый, спокойный, добрый, уютный, размеренный.

## 2. Цель этой сессии

Нужно отдельно проработать браузерную игру внутри Chrome new-tab extension.

Это НЕ Steam-игра, НЕ нижняя desktop-полоска и НЕ sidescroll factory.

Это top-down / top-view idle farm, открывающаяся на новой вкладке браузера.

Задача будущего чата: придумать основу игры, которая:  
- живёт в new tab;  
- не мешает искать информацию;  
- имеет строку поиска;  
- имеет большой блок под рекламу;  
- имеет уютную idle-ферму;  
- играет сама, даже если пользователь не сидит в ней постоянно;  
- даёт пользователю приятные короткие поводы возвращаться;  
- связывает рекламный просмотр/показ с отправкой помощи собакам этично и прозрачно.

## 3. Базовая форма продукта

Пользователь открывает новую вкладку Chrome.

На экране:  
- строка поиска;  
- крупный рекламный блок / sponsorship block;  
- top-down ферма;  
- собаки;  
- грядки;  
- постройки;  
- производство корма;  
- отправки помощи в приюты;  
- лёгкие задания и улучшения.

Формат: что-то в духе сильно упрощённой, более idle и более спокойной Stardew Valley-like фермы.

Важно: игрок НЕ обязан сидеть и играть. Игра должна “играть за него”. Пользователь иногда приходит, помогает, строит, выбирает улучшения, поднимает уровни собачкам и постройкам.

## 4. Отличие от Steam/Desktop игры

Steam/Desktop:  
- sidescroll нижняя always-on-top полоска;  
- интерфейсная ДНК Crusaders Quest: Hero Town;  
- возможно, больше похоже на горизонтальный заводик/кооператив;  
- рекламной монетизации нет.

Browser Extension:  
- top-down ферма на новой вкладке;  
- больше пространства для классической фермы;  
- можно реализовать грядки, тропинки, здания, животных, декор;  
- рекламная модель здесь уместна как core business model;  
- нужно очень аккуратно встроить рекламу в charity loop.

## 5. Главный fantasy продукта

“Я открываю новую вкладку, а там живёт маленькая спокойная собачья ферма. Она сама растёт, собаки сами ходят, что-то делают, корм готовится и отправляется в приюты. Я могу задержаться на 20 секунд, помочь, улучшить что-то и уйти дальше.”

## 6. Возможный core loop

Короткий визит:  
1. Пользователь открывает новую вкладку.  
2. Видит ферму и текущий прогресс.  
3. Реклама/спонсорский блок засчитывает безопасный просмотр или предлагает добровольное действие.  
4. На ферме копится “помощь” / “sponsor food points” / ресурс отправки.  
5. Пользователь собирает результаты, назначает улучшение, отправляет фургон или помогает одной собаке.  
6. Закрывает вкладку или ищет через search bar.

Длинный визит:  
1. Игрок настраивает план фермы.  
2. Улучшает постройки.  
3. Размещает декор.  
4. Открывает новых собак.  
5. Настраивает production priorities.  
6. Читает отчёты/истории собак.

Offline/idle:  
- ферма продолжает производить в пределах мягкого cap;  
- нет наказания за отсутствие;  
- возвращение приятно, а не тревожно.

## 7. Реклама и этика

В браузерной версии рекламная монетизация допустима и, вероятно, является основной бизнес-моделью.

Но она должна быть:  
- прозрачной;  
- добровольной там, где есть активные rewarded-действия;  
- не манипулятивной;  
- без guilt pressure;  
- без “посмотри рекламу, иначе собаки голодают”;  
- без блокировки core gameplay;  
- с понятным отчётом, что засчитывается и как это связано с помощью.

Правильный тон:  
“Спасибо, этот просмотр помогает отправлять корм.”

Неправильный тон:  
“Если ты не посмотришь рекламу, собаки останутся голодными.”

Важно: Chrome Web Store policies, рекламные SDK, privacy, consent, personal data и благотворительная отчётность требуют отдельной юридической/технической проверки. Будущий чат может формулировать UX-гипотезы, но не должен выдумывать юридические гарантии.

## 8. Игровая структура

Так как это top-down ферма, здесь можно использовать более классические системы:

- грядки;  
- полив;  
- сбор;  
- переработка;  
- кухня/кормоцех;  
- склад;  
- фургон отправки;  
- домики собак;  
- декор;  
- дерево исследований;  
- уровни собак;  
- уровни построек;  
- мягкие daily/weekly goals;  
- коллекция собак.

Но даже здесь игра должна быть idle:  
- собаки сами ходят и работают;  
- пользователь задаёт приоритеты;  
- микроменеджмент минимальный;  
- нет наказания за невнимание;  
- нет агрессивного FOMO.

## 9. Собаки

Собаки — главный эмоциональный центр.

У каждой собаки может быть:  
- имя;  
- порода/тип;  
- характер;  
- роль на ферме;  
- любимое занятие;  
- уровень;  
- навыки;  
- комната/домик;  
- история;  
- возможно, связь с реальной собакой/приютом.

Породистые собаки допустимы: такса, корги, помски, лабрадор, шпиц, хаски и т.д.

Приютские/реальные собаки тоже допустимы, но подаются без давления и без эксплуатации жалости.

Не делать разведение собак как core mechanic. Коллекционирование жителей — да. Breeding ради редкости — рискованно и пока не нужно.

## 10. Производство корма

Главная задача фермы: делать корм и отправлять его в приюты.

Мирные ресурсы:  
- овощи;  
- крупы;  
- рис;  
- овёс;  
- травы;  
- ягоды;  
- вода;  
- витамины;  
- упаковка.

Белковая часть:  
- протеин-пакеты;  
- бульонные капсулы;  
- поставки партнёров;  
- ветеринарная смесь.

Запрещено:  
- скотобойня;  
- разделка;  
- кровь;  
- реалистичная мясная цепочка;  
- собаки, которые убивают животных.

Главная задача: не сделать “полноценную игру, которая случайно лежит в новой вкладке”, а сделать именно new-tab idle companion, где каждый короткий визит приятен, полезен и не мешает основной функции браузера.

## 14. Approved core loop / D-008

Дата: 2026-06-24.

Approved-вариант: “Спонсорская ферма → производство корма → отправка фургона в приют”.

Базовая формула:  
Открыл вкладку → увидел живую ферму и спонсорский блок → собаки произвели корм → пользователь одним коротким действием собирает, помогает, улучшает или отправляет фургон → фургон увозит помощь → ферма становится чуть уютнее.

Основной core loop:  
1. Пользователь открывает новую вкладку.  
2. Видит строку поиска, sponsorship/ad block и живую top-down ферму.  
3. Собаки за время отсутствия вырастили мирные ресурсы, отнесли их в кормоцех, подготовили часть корма или упаковки.  
4. Спонсорский блок этично добавляет вклад в ресурс отправки / sponsor food points / фонд отправки.  
5. Пользователь за 5–20 секунд делает одно необязательное действие: собирает результат, помогает собаке, выбирает улучшение, задаёт приоритет или отправляет фургон.  
6. Фургон уезжает в приют; игрок видит спокойный отчёт без давления и guilt.  
7. Игрок закрывает вкладку или пользуется search bar.

Слои поверх core loop:  
- Целевой слой: мягкие заявки приютов — например, овощные пайки, витаминная смесь, упаковка, крупы.  
- Эмоциональный слой: собаки с именами, характерами, ролями, любимыми занятиями, домиками и навыками.  
- Долгосрочный слой: визуальный рост фермы — новые грядки, кормоцех, склад, фургонная площадка, декор, домики, дорожки, зоны.  
- Микро-визиты: каждое открытие вкладки может дать одно мягкое необязательное действие, но без серии, таймеров наказания и агрессивного FOMO.

Реклама и благотворительность:  
Рекламный блок не должен блокировать core gameplay. Активные rewarded-действия должны быть добровольными. Запрещены формулировки вроде “если ты не посмотришь рекламу, собаки останутся голодными”. Правильный тон: “Спасибо, этот просмотр помогает отправлять корм.” Точную связь денег, показов, помощи, отчётности, privacy, consent, рекламных SDK и Chrome Web Store policies нужно отдавать в отдельную юридическую/техническую проверку.

Статус: принято пользователем как основной core loop browser-extension ветки и зафиксировано в `02_DECISIONS` как D-008.

## 15. Shared world role / D-012

Дата фиксации: 2026-06-25.

Browser Extension теперь дополнительно трактуется как настоящая ферма общего мира Shelter. Это не Steam-полоска и не sidescroll factory: это top-down new-tab idle farm, где выращиваются, собираются и подготавливаются ресурсы.

В общем мире Browser Farm может быть местом, откуда ресурсы уходят в большой собачий мир и narrative-связью поддерживают Steam/Desktop кооператив/мастерскую. Steam/Desktop, в свою очередь, показывает кооператив, куда ресурсы приезжают и где из них делают корм, аксессуары, одежду, игрушки, лежанки, пледы, наборы помощи и отправки в приюты.

MVP-правило: связь со Steam/Desktop только narrative-only. Browser Extension не должен требоваться для прогресса в Steam, а Steam не должен быть обязательным для Browser Extension. Нельзя превращать связь продуктов в давление или обязательную установку второго продукта.

Advertising/sponsorship остаются допустимыми только в рамках D-008: реклама не блокирует core gameplay, не использует guilt pressure и не формулируется как “посмотри рекламу, иначе собаки пострадают”. Все privacy, ads SDK, Chrome Web Store, sponsorship disclosure и charity claims требуют отдельной проверки.

Статус: D-012 принято пользователем и зафиксировано в `02_DECISIONS`.
