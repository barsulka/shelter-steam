## 3. Capture command

Add or update a whitelisted/dev command in `steam/tools/dev-vertical-slice.sh`.

Preferred command:

```sh
cd steam
tools/dev-vertical-slice.sh first-day-visible-capture
```

Acceptable alternative if it fits existing style better:

```sh
cd steam
tools/dev-vertical-slice.sh capture-first-day-mvp
```

The command should:

1. run visible macOS Godot capture mode, not headless;
2. use current First Day MVP runtime polish implementation;
3. produce named screenshots and frame sequence under the persistent output path;
4. also include state proof artifacts for the same captured loop;
5. not require manual browser interaction;
6. not mutate gameplay contracts;
7. not create final visual acceptance claims.

If adding a new helper command requires MCP whitelist follow-up, document that in `CODEX_STATUS.md`; do not modify the sibling MCP repo unless this brief explicitly expands to that repo. This brief is primarily for the Shelter/Steam repo.

## 5. Required frame sequence

Create a PNG-frame sequence:

```text
captures/video/first_day_mvp_visible_loop_frames/frame_0001.png ...
```

Target:

- enough frames to inspect the full first-day loop visually;
- not necessarily MP4;
- frame sequence fallback is acceptable.

Preferred options:

1. Low-speed deterministic capture that covers the full loop with enough frames to inspect state transitions.
2. If full real-time capture is too long, use a staged capture sequence with representative moments and document it clearly.

Do not claim real player-feel if the sequence is accelerated.

## 9. What to check visually

The capture pack should make it possible to answer:

1. Are dogs visible as characters rather than tiny anonymous units?
2. Is the route departure/return readable?
3. Is payload arrival readable before storage changes?
4. Are unload/carry actions readable?
5. Is Kitchen -> Food Mix readable?
6. Is Food Mix -> Packing Table readable?
7. Is Packing Table -> Food Bag readable?
8. Is Food Bag -> Van readable?
9. Is the dispatch confirmation state calm and clear?
10. Does postcard/reward feel like a warm closure or only a UI flag?
11. Is Dog Card memory/slippers separation visible enough for D-010?
12. Is the next-day hint gentle, not pressure?
13. Does compact UI dominate the strip or support it?
14. Do 216/144/96 previews remain readable enough?

Codex should not answer these as final judgement. Codex creates the evidence pack.
