## 3. Board structure to keep

A good Steam overlay system board may include:

1. **Header**  
   - project/product name;  
   - pack name;  
   - asset list;  
   - current status.

2. **Approved main strip reference**  
   - small screenshot of `D-011_steam_overlay_main_strip_v1_reference.png`;  
   - short note: bottom-hugging overlay, empty top, dogs and actions first.

3. **Overlay principles short block**  
   - lives at the bottom;  
   - top is empty / transparent / black;  
   - functional silhouettes;  
   - dogs and actions before decor;  
   - 144 px working minimum;  
   - no interior cutaways in main overlay.

4. **Asset cards**  
   Each asset card must show:  
   - asset name;  
   - asset class: Building / Utility Prop / Dog Action Sprite;  
   - function;  
   - readable silhouette;  
   - icon / plaque if needed;  
   - dog action;  
   - pass/fail notes.

5. **Integration example**  
   - asset inserted into a low-profile main strip;  
   - no full UI;  
   - no big floating windows;  
   - clear spacing between modules.

6. **Composition variations**  
   - 2–3 layouts showing different rhythm;  
   - modules and utility props must not merge into dense village.

7. **Palette block**  
   - warm wood;  
   - muted greens;  
   - cream highlights;  
   - soft shadow/dark background;  
   - dog accent colors.

8. **Materials / textures block**  
   - wood;  
   - stone;  
   - fabric sacks;  
   - paper plaques;  
   - metal pipes/buckets;  
   - grass/leaves.

9. **Key rule reminder**  
   - clear silhouette;  
   - dog first;  
   - function readable;  
   - 144 px working minimum;  
   - peaceful / warm tone.

10. **Scale check**  
   - 216 px comfort;  
   - 144 px working minimum;  
   - 96 px stress test.

## 4. New mandatory taxonomy rules for boards

Every board must label each asset as one of:

- **Building**  
- **Utility Prop**  
- **Dog Action Sprite**

Hard rule:

> Utility Props must not become houses.

If a board presents a Utility Prop as a cottage, shed, room, or decorative village building, that board fails.

## 6. Prompt for future system board generation

Use this as a base prompt when creating a new structured board:

```text  
Create a structured visual system board for Shelter Steam Overlay Main Strip.

Board purpose: present approved / candidate overlay assets as an art-direction system, not final production exports.

Include: header, approved main strip reference, overlay principles, asset cards, integration example, composition variations, palette, materials/textures, key rule reminder, scale check 216/144/96.

Use current D-011 rules: bottom-hugging overlay, huge empty top, functional silhouettes, dogs and actions first, no interior cutaways in main overlay, no art nouveau, no fantasy village, no aggressive UI, no combat/horror/meat/guilt imagery.

Every asset card must show asset class: Building / Utility Prop / Dog Action Sprite.

Hard taxonomy rule: Utility Props must not become houses. Buildings are rare anchors. Dog Action Sprites are a separate meaning layer.

Use Approved Library v1 style and content. Do not use rejected Asset Pack 1 v1 asset designs as approved content.  
```
