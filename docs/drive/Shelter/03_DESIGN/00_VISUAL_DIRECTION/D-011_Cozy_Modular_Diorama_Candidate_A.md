# D-011 — Visual Direction Candidate A: Cozy Modular Diorama

Дата: 2026-06-25  
Роль документа: Visual Direction / Art Direction  
Статус: Candidate A принят для проверки, style board и readability test. Не является финальным визуальным стилем навсегда.

## 1. Decision summary

Для Shelter принимаем Cozy Modular Diorama как основное визуальное направление-кандидат.

Формула направления:

Уютная модульная собачья диорама — маленький кооператив из комнат, теплиц, кухонь, складов, домиков и фургонов, где собаки-жители медленно и видимо делают доброе общее дело.

Это направление должно объединять две продуктовые формы:

1. Steam/Desktop: side/cutaway модули для горизонтальной always-on-top production strip.  
2. Browser Extension: top-down / 3⁄4 ферма для Chrome new-tab idle farm.

Важно: направление принято как кандидат для проверки, а не как окончательный production style. До финального утверждения оно должно пройти style board, readability test, оценку asset scope и проверку совместимости Steam + Browser.

## 2. Почему выбран именно этот кандидат

Cozy Modular Diorama лучше всего соединяет:

- Steam core: горизонтальный собачий производственный кооператив, cozy idle production strip + dog community sim.  
- Browser core: top-down спонсорская ферма, производство корма и отправка фургона в приют.  
- Эмоциональное ядро: собаки как жители и персонажи.  
- Референсы: Crusaders Quest: Hero Town для модульной полоски и скрываемого UI, Rusty’s Retirement для медленных физических действий, Yog-Sothoth’s Yard для комнат и желания украшать личные пространства.

Кандидат не требует превращать Shelter ни в чистый pixel art, ни в дорогую иллюстративную storybook-игру, ни в корпоративное vector-приложение.

## 3. Что берём из референсов

### Crusaders Quest: Hero Town

Берём:  
- горизонтальную ленту / поле на всю ширину экрана;  
- деление на зоны и блоки;  
- ощущение маленьких модулей;  
- возможность скрывать интерфейс;  
- минимализм и прозрачность для Steam/Desktop.

Не берём:  
- автобаттлер;  
- боевую прогрессию;  
- призыв монстров;  
- PvP, боссов, арены;  
- агрессивную hero-collection экономику.

### Rusty’s Retirement

Берём:  
- медленный rhythm наблюдения;  
- персонажи физически идут ногами к задаче;  
- перенос предметов, мешков, коробок, воды, рулонов, инструментов;  
- строительство, декор и перенос объектов не мгновенные;  
- удовольствие от наблюдения, а не микроменеджмент.

Не берём:  
- перегруженность фермерской симуляцией;  
- любой pressure loop, наказание за отсутствие или агрессивный FOMO.

### Yog-Sothoth’s Yard

Берём:  
- комнатность;  
- уютные маленькие пространства;  
- ощущение “хочу украсить комнату”;  
- комнату как витрину характера жителя.

Не берём:  
- хоррор / occult настроение;  
- мрачную красно-чёрную драму;  
- давящий визуальный тон;  
- любые мотивы, конфликтующие с добрым Shelter.

## 4. Базовый визуальный язык

### Общая форма

- 2D-диорама с крупными читаемыми силуэтами.  
- Мягкий рисованный контур, но не чрезмерная детализация.  
- Тёплые материалы: дерево, ткань, керамика, зелень, бумага, картон, мешковина, мягкий свет.  
- Предметы выглядят аккуратно и мирно, особенно всё, что связано с кормом.  
- Мир должен ощущаться handmade / cozy, но не игрушечным и не инфантильным.

### Цвет

- Базовая палитра: тёплые нейтрали, зелёные фермерские тона, мягкий кремовый, древесные оттенки.  
- Акценты: кормовые мешки, фургон, собачьи аксессуары, UI-стикеры, sponsor card.  
- Избегать: кислотных free-to-play цветов, тревожного красного как основного сигнала, мрачного хоррор-контраста.

### Свет

- Мягкий дневной свет для фермы.  
- Тёплый комнатный свет для собачьих домиков.  
- Никакой драматичной тьмы, резких хоррор-теней и ощущения опасности.

## 5. Steam/Desktop projection: side/cutaway

Steam-версия использует side/cutaway модули.

Главные правила:

- Модуль должен читаться по силуэту: теплица, кухня, фасовка, склад, комната, фургонная площадка.  
- В малой высоте приоритет у действия, а не у декора.  
- В пустых местах фон может быть прозрачным или очень спокойным, чтобы не мешать работе пользователя.  
- UI должен быть скрываемым и минимальным.  
- Комнаты и production modules должны быть отделены визуально: комната — личное пространство, модуль — рабочая зона кооператива.

Минимальный набор Steam test modules:

1. Комната собаки.  
2. Теплица / садовый модуль.  
3. Кухня / кормоцех.  
4. Миксер / сушилка / фасовка.  
5. Склад мешков.  
6. Фургонная площадка.

## 6. Browser Extension projection: top-down / 3⁄4

Browser-версия использует top-down / 3⁄4 ферму.

Главные правила:

- New tab сначала должен оставаться удобной новой вкладкой: search bar не спорит с фермой.  
- Sponsorship/ad block интегрируется как спокойная sponsor card / доска партнёра, но не маскируется под core gameplay.  
- Ферма должна быть приятна для короткого взгляда: 5–20 секунд.  
- Собаки должны быть видны и узнаваемы даже без zoom.  
- Грядки, дорожки, домики, кормоцех и фургон должны считываться без объяснений.

Минимальный набор Browser test modules:

1. Search bar.  
2. Sponsor card.  
3. Фермерское поле / грядки.  
4. Домики собак.  
5. Кормоцех.  
6. Склад / площадка отправки.  
7. Фургон.  
8. Мягкий отчёт отправки.

## 7. Dog shape system

Собака должна читаться в порядке:

силуэт → порода/тип → действие → настроение → индивидуальная деталь.

Базовые параметры породы / типа:

- размер тела;  
- длина лап;  
- форма головы;  
- уши;  
- хвост;  
- шерсть / пушистость;  
- окрас и пятна;  
- походка;  
- 1–2 характерные idle-анимации.

Первые породы / типы для проверки:

1. Корги — низкий, плотный, большие уши.  
2. Такса — длинное тело, короткие лапы.  
3. Шпиц / померанский — маленький пушистый шар, хвост-кольцо.  
4. Хаски — острые уши, хвост, контрастная маска.  
5. Лабрадор — крупный мягкий силуэт.  
6. Дворняжка — смешанный тип с уникальным окрасом.  
7. Маленькая пушистая собака.  
8. Крупная спокойная собака.

Правило: порода не должна быть только перекраской. Если собаки отличаются только цветом, стиль не прошёл проверку.

## 8. Room-as-personality rules

Комната собаки — главный визуальный носитель личности.

Комната может показывать:

- характер;  
- любимую работу;  
- привычки;  
- любимый декор;  
- мягкую привязанность игрока к собаке;  
- маленькие бытовые истории;  
- следы развития собаки.

Комната не должна становиться обязательной min-max системой.

Запрещённый вайб:

- “без легендарных обоев собака неэффективна”;  
- “комнату нужно оптимизировать как билд”;  
- “собака — контейнер для статов и баффов”.

Допустимый вайб:

- “эта собака любит тёплый коврик и садовые инструменты”;  
- “эта собака аккуратная, у неё всё по полочкам”;  
- “эта собака смешная, она собирает тапочки”;  
- “эта собака любит фасовку, поэтому у неё маленькая полка с бумажными ярлыками”.

Мягкие бонусы допустимы, но они должны усиливать индивидуальность, а не заменять её.

## 9. Production design: корм без визуальной жестокости

Производство корма должно быть мирным, чистым и уютным.

Разрешённые визуальные ингредиенты:

- овощи;  
- тыква;  
- рис;  
- овёс;  
- травы;  
- ягоды;  
- вода;  
- витаминные баночки;  
- упаковка;  
- протеин-пакеты;  
- бульонные капсулы;  
- ветеринарная смесь;  
- поставки партнёров в аккуратных коробках.

Запрещено:

- туши;  
- кровь;  
- ножи как производственный символ;  
- разделка;  
- скотобойня;  
- реалистичная мясная цепочка;  
- собаки, которые убивают животных.

## 10. UI principles

UI должен быть спокойным, минимальным и не давящим.

Steam UI:

- скрываемый;  
- полупрозрачный;  
- не мешает рабочему столу;  
- показывает зоны, задачи, здание, собаку, очередь, отправку;  
- не превращается в мобильную free-to-play панель.

Browser UI:

- search bar имеет высокий приоритет;  
- sponsor card видна, но не агрессивна;  
- ферма не заслоняет основную функцию новой вкладки;  
- отчёты помощи спокойные, без guilt pressure;  
- кнопки коротких действий ясные и необязательные.

Правильный тон sponsor card:

“Спасибо, этот просмотр помогает отправлять корм.”

Неправильный тон:

“Если ты не посмотришь рекламу, собаки останутся голодными.”

## 11. Style board scope

Style board должен включать 12–20 референсов / слотов:

1. Собаки: силуэты и породы.  
2. Собаки: выражения и idle-позы.  
3. Cutaway-комнаты.  
4. Комнаты как характер персонажа.  
5. Маленькие production modules.  
6. Теплица / сад.  
7. Кухня / кормоцех.  
8. Фасовка / склад.  
9. Top-down ферма.  
10. Дорожки, грядки, домики.  
11. UI cards / мягкие панели.  
12. Sponsor card / доска партнёра.  
13. Фургон и отправка.  
14. Мирные ингредиенты.  
15. Материалы: дерево, ткань, бумага, зелень.  
16. Масштаб и читаемость.  
17. Что не брать: мрак, хоррор, guilt, exploitation.

## 12. Readability test gate

Стиль нельзя финализировать, пока он не прошёл readability test в Steam-полоске на высотах:

- 96 px — супер-минимум;  
- 144 px — вероятный рабочий минимум;  
- 216 px — комфортный режим.

Тестовые сцены:

1. Steam 01: собака несёт мешок корма из фасовки на склад.  
2. Steam 02: собака переклеивает обои / украшает комнату.  
3. Steam 03: модуль “кухня → миксер → фасовка” в одной горизонтальной цепочке.  
4. Browser 01: top-down ферма с грядками, дорожками, 3–5 собаками и кормоцехом.  
5. Browser 02: new-tab layout: search bar, sponsor card, ферма, маленький отчёт отправки.

Pass criteria:

- собака узнаётся как собака;  
- действие читается за 1 секунду;  
- предмет в переноске читается: мешок, коробка, лейка, рулон обоев, инструмент;  
- производственный модуль отличим от комнаты;  
- комната собаки выглядит личной, но не перегруженной;  
- Browser layout не ощущается рекламным сайтом;  
- sponsor card выглядит спокойно и отдельно от core gameplay;  
- стиль допускает много пород без полной перерисовки системы.

Fail criteria:

- стиль красивый только на больших скриншотах;  
- в Steam-полоске всё превращается в кашу;  
- комнаты слишком детальные и дорогие;  
- собаки выглядят как одинаковые mascots с разным окрасом;  
- производство выглядит как фабрика эксплуатации;  
- sponsor block выглядит как баннер, который вторгся в игру;  
- декор комнаты обещает обязательную силу.

## 13. Production risks

### Risk 1 — две проекции удваивают scope

Решение: на MVP не все ассеты обязаны иметь обе версии. Для ключевых собак, зданий и материалов проектируем общую идентичность, но production выполняем по приоритетам.

### Risk 2 — диорама может потерять читаемость в Steam

Решение: ранний readability test на 96 / 144 / 216 px до финализации стиля.

### Risk 3 — комнаты могут стать слишком дорогими

Решение: модульная система комнат: пол, стены, коврик, лежанка, 1–2 любимых предмета, 1 personality object.

### Risk 4 — собаки могут выглядеть как рабочие-эксплуатируемые

Решение: показывать кооператив и личность: отдых, игра, домики, добровольная занятость, юмор, забота. Избегать заводской суровости.

### Risk 5 — sponsor card может выглядеть как агрессивный баннер

Решение: отделить sponsor card от core gameplay, визуально сделать её спокойной доской партнёра, запретить guilt-formulations.

### Step 5 — Producer gate

Вернуться в продюсерский штаб с результатом:

- pass/fail по читаемости;  
- оценка production scope;  
- корректировки Candidate A;  
- решение: финализировать, сузить или пересобрать стиль.

## 16. Addendum — Steam Overlay Main Strip v1

Дата: 2026-06-25.

После просмотра CQ Hero Town overlay и нового visual pass для Steam/Desktop в D-011 добавляется уточнение: Steam/Desktop использует не один общий side/cutaway-формат, а три визуальных режима.

1. **Main overlay strip** — главный always-on-top режим. Это нижняя живая кромка экрана: пустота сверху, маленькие функциональные фасады внизу, крупные собаки и читаемые действия. Интерьерная детализация запрещена.  
2. **Inspect / detail views** — открываемые по клику просмотры зданий и комнат. Здесь допустимы cutaway-интерьеры, амбар изнутри, полки, мешки, комната собаки, обои, декор и personality details.  
3. **Windowed / scenic mode** — полноценное окно игры с фоном, небом, горами, водой/отражением и более атмосферной сценой.

Approved reference: `D-011_steam_overlay_main_strip_v1_reference.png` в `03_DESIGN/04_DELIVERABLES/approved_art_files/`.

Main overlay rule:

> Steam Overlay Main Strip is not a small version of the inspect view. It is a dedicated low-profile layer: functional silhouettes + big readable dogs + peaceful actions + empty top space.

## 17. Addendum — Asset taxonomy for Steam overlay

Дата: 2026-06-25.

После Asset Pack 1 v1 фиксируется дополнительное правило для всех Steam overlay ассетов.

Перед генерацией любой сущности нужно сначала определить её тип:

1. **Building / здание.** Редкий крупный якорь main strip: dog house, kitchen, storage, delivery van endpoint. Здания могут иметь фасад, крышу и дверь, но остаются низкими и функциональными.  
2. **Utility Prop / функциональный объект.** Вспомогательный объект или пауза между зданиями: мельница, водяная станция, насос, бак, тележка, компост, указатель, фонарь. Utility Props запрещено превращать в домики.  
3. **Dog Action Sprite / действие собаки.** Собака с крупным читаемым объектом: несёт мешок, везёт тележку, поливает, красит доску, клеит ярлык. Это отдельный смысловой слой, а не декор.

Главное правило:

> В main overlay здания — редкие якоря. Всё остальное — маленькие функциональные объекты и действия собак. Если каждая функция превращается в домик, нижняя полоса снова становится плотной декоративной деревней и нарушает D-011.

Asset Pack 1 v1 получает статус: useful exploration, not approved production pack. Приняты: тон, палитра, материалы, dog action direction, icon plaques. Не приняты как production assets: текущие версии мельницы, водяной станции и мастерской декора, потому что они слишком легко превращаются в отдельные домики.
